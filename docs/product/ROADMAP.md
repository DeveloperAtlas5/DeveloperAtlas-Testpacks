# Public roadmap

Developer Atlas is narrowing around a single outcome: helping developers remain
in control of AI-assisted code changes.

## Now

- validate the 15-minute public Control exercise with people outside the project;
- complete the same-tester external-alpha retest;
- finish clean-profile installation and UX hardening for Control Lite in Navigator;
- reduce Navigator information density through progressive disclosure;
- review and publish the checksum-listed limited Navigator preview and demo workspace;
- validate the runnable Control exercise and its CI evidence with external users;
- complete independent human review for the highest-value knowledge nodes.

## Next

- manually deploy the allowlisted six-node Compendium preview to HTTPS after release review;
- run a 10–20 person independent evaluation;
- measure activation, completed controlled changes, repeat use, and review time;
- test the same bounded task across multiple AI tools;
- obtain credible team-level willingness-to-pay evidence before building billing;
- complete accessibility and cross-platform release checks.

## Later, only if evidence supports it

- shared team conventions and decision memory;
- signed organization knowledge and policy packs;
- multi-project history;
- managed onboarding or private deployment;
- broader framework support.

## Parallel supporting research

- validate Continuity on a real Windows Laravel/Vue/SQLite project;
- extend First Expedition only where it improves Control or Navigator onboarding;
- complete the Vue learning track before presenting it as equivalent to the browser and Laravel paths.

## Explicit non-goals

Atlas is not trying to become an autonomous coding agent, a general-purpose IDE,
a surveillance system, or a guarantee that AI-generated code is correct.

Roadmap items are directions, not release promises. Evidence from real workflows
can change their order.
