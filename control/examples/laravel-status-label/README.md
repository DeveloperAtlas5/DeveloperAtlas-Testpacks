# Worked example: bound one Laravel status-page change

This is the complete artifact set used by the [15-minute Atlas exercise](../../../TRY_ATLAS.md).

| Artifact | Purpose |
| --- | --- |
| [STARTING_POINT.md](STARTING_POINT.md) | The route, controller, and Blade code before the request |
| [BASELINE_PROMPT.md](BASELINE_PROMPT.md) | A plausible but unbounded request |
| [ATLAS_INSTRUCTION.md](ATLAS_INSTRUCTION.md) | The same request with scope, acceptance, and verification boundaries |
| [PROPOSED_PATCH.diff](PROPOSED_PATCH.diff) | A small proposed implementation to review |
| [VERIFICATION_CHECKLIST.md](VERIFICATION_CHECKLIST.md) | Human evidence checklist |
| [SAMPLE_EVIDENCE.md](SAMPLE_EVIDENCE.md) | Honest repository-only evidence result |
| [DECISION_RECORD.md](DECISION_RECORD.md) | Completed human revision decision |
| [COMPLETED_EXAMPLE.md](COMPLETED_EXAMPLE.md) | Why revision is a complete and honest loop outcome |

The code is a teaching fragment, not a runnable Laravel application. No model was benchmarked and no
Laravel test command was executed for this static exercise. The example demonstrates the decision
process, including the valid outcome “revise until runtime evidence exists.”
