# Developer Atlas Pack Ladder

Developer Atlas uses small packs so feedback stays focused.

Each pack is designed for a different starting point.  
Testers should choose one path, not the whole repo.

The goal is simple:

```txt
open one pack
understand what to do next
try one focused thing
stop when something feels unclear
send useful feedback
Public paths
1. Beginner Pack

Folder:

packs/beginner/
Best for

The Beginner Pack is for people who are new to web development, or who are not yet sure how HTML, CSS, and JavaScript fit together.

This is the safest first step for absolute beginners.

What it contains

The pack includes a tiny web project with three files:

index.html
style.css
script.js

It also includes calm beginner lessons explaining:

what HTML does
what CSS does
what JavaScript does
how the files connect
how to open an HTML file in a browser
how to make one small change and see it work
What testers will try

Testers will open a tiny web page, make one small text/style/behavior change, refresh the browser, and see the result.

Current status

Public test pack.

Main test question
Are the first-start instructions clear enough for someone new to web projects?
Send this pack to people who
have never built a web page
do not know what HTML/CSS/JavaScript each do
are not ready for Vue yet
need a very small first win
2. Rookie Pack

Folder:

packs/rookie/
Best for

The Rookie Pack is for people who already have a basic Vue project, or people who are ready to build a simple Vue to-do app.

This pack is not for absolute first-time web users.
It assumes the tester can open a project and follow one small mission.

What it contains

The pack includes one focused Vue mission:

Make a to-do list remember items after refresh.

It includes:

a calm mission dashboard
a START_HERE.md
one guided mission
simple recovery/help material
a feedback path
What testers will try

Testers will follow one Vue mission and check whether the app remembers to-do items after refreshing the page.

Current status

Public test pack.

Main test question
Can the tester complete one calm Vue mission without getting overwhelmed?
Send this pack to people who
know basic HTML/CSS/JavaScript
have touched Vue before
have a simple Vue project
are ready for a guided coding mission
need a small practical win
3. Library Pack

Folder:

packs/library/
Best for

The Library Pack is for people who want quick explanations of programming terms without reading long documentation.

It is useful when a tester gets stuck on a word, concept, file type, or AI instruction.

What it contains

The pack includes small concept cards for topics like:

HTML page structure
CSS page style
JavaScript page behavior
browser vs editor
file paths
terminal commands
Vue components
Vue state
localStorage
AI explain vs patch
scope creep
small patches

Each card is intentionally short.

Card format

Each card explains:

what the term means
when to use it
a tiny example
what to watch out for
What testers will try

Testers will open one or more concept cards and judge whether the explanation helps them continue.

Current status

Public test pack.

Main test question
Do concept cards help testers understand terms quickly enough to keep going?
Send this pack to people who
get stuck on programming words
want short explanations
dislike long documentation
are learning with AI and need clearer terms
need a calm reference while working
4. AI Collaboration Pack

Folder:

packs/ai-collaboration/
Best for

The AI Collaboration Pack is for people who use AI coding tools and want better control.

It is especially useful for people who feel that AI:

changes too much
writes code too quickly
skips explanations
adds features they did not ask for
makes them feel less in control
What it contains

The pack teaches the Atlas AI action model:

Explain
Suggest
Patch
Confirm
Park
Refuse

It includes:

an action model guide
prompt templates
beginner-safe code review checklist
examples of good AI control
feedback template
What testers will try

Testers will use one prompt with their AI tool and check whether the action model helps them stay in control.

Current status

Public test pack.

Main test question
Do guardrails help AI assist without taking over?
Send this pack to people who use
ChatGPT
Claude
Cursor
GitHub Copilot
Codex
Aider
Continue-style tools
other AI coding assistants
Send this pack to people who
already use AI for coding
want better prompts
want AI to explain before changing code
want small patches instead of big rewrites
want to stop scope creep
want to review AI output more safely

Why this ladder exists

Different people get stuck in different places.

A beginner may need help opening a project.

A rookie may need one concrete mission.

A builder may need quick explanations.

An AI-assisted coder may need scope control.

The pack ladder helps people choose the right surface instead of seeing the entire system at once.

Recommended tester routing
Tester type	Send them
Never built a web page	Beginner Pack
Does not understand HTML/CSS/JavaScript yet	Beginner Pack
Knows basic HTML/CSS/JavaScript	Beginner Pack or Rookie Pack
Has a basic Vue project	Rookie Pack
Wants one calm coding mission	Rookie Pack
Gets confused by programming terms	Library Pack
Wants quick reference cards	Library Pack
Uses AI for coding	AI Collaboration Pack
AI often does too much	AI Collaboration Pack
Feels overwhelmed easily	Beginner Pack or Library Pack
What feedback to ask for

After testing one pack, ask:

What did you open first?
Did you know what to do next?
What felt clear?
What felt confusing?
Did it feel calm or overwhelming?
Where did you stop?
Would you try another pack?
Feedback rule

Ask testers to stop when something feels unclear.

Confusion is useful feedback.

The tester is not being tested.
The pack is being tested.
